# Docs Sync

Update architecture, governance, and harness docs when the canonical catalog changes.

## Triggers

- changing canonical agent specs
- changing exported Codex or Antigravity surfaces
- changing lifecycle or verification logic

## Checklist

- update architecture docs with current implementation
- update guidance files that point at moved truths
- avoid planned-only language when code has landed

## Outputs

- doc updates
- truth-source adjustments
- task-tracker sync
