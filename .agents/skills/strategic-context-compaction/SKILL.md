# Strategic Context Compaction

Compact context at stable phase boundaries instead of mid-implementation.

## Triggers

- after audit is complete
- after a migration milestone lands
- before switching from architecture to implementation

## Checklist

- write down the stable plan before compacting
- keep active file and artifact references
- avoid compacting mid-fix

## Outputs

- phase checkpoint
- preserved references
- next-step summary
