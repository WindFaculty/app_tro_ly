# UI - Current Implementation And Planned Direction

Updated: 2026-03-26

This document separates the implemented Unity UI from planned design direction.

## Current Implementation

### Runtime entry

- `Assets/Resources/UI/MainUI.uxml` is the UI Toolkit entrypoint loaded at runtime.
- The visible shell comes from `Assets/Resources/UI/Shell/AppShell.uxml`.
- Runtime styles come from `Assets/Resources/UI/Styles/*.uss`.

### Top-level shell

The current shell has:

- a top bar
- a left sidebar
- a center-stage content area
- a right-side interaction area
- a subtitle overlay
- a reminder overlay

This is already a three-region UI Toolkit shell. It is not a future-only concept.

### Current screens

#### Home

- Shows a placeholder avatar card
- Shows a quick-add text field and quick-add button
- Shows task summary text and a text-based today snapshot
- Uses placeholder copy for the avatar area rather than a finished production avatar scene

#### Schedule

- Shows week, inbox, and completed routing through `AppRouter`
- Uses a placeholder calendar panel that currently renders text summaries rather than a full calendar grid
- Shows a static schedule-side assistant panel in the right column

#### Settings

- Shows backend-backed settings summary
- Exposes toggles for:
  - speak replies
  - transcript preview
  - mini assistant enabled flag
  - reminder speech enabled flag
- Exposes reload and save actions

#### Chat panel

- Shows transcript text
- Supports text input
- Supports send
- Supports mic toggle
- Shows transcript preview and route diagnostics through the backing view-model store

### Overlays

- `SubtitleOverlay.uxml` renders temporary subtitle text during or after speech output.
- `ReminderOverlay.uxml` renders in-app reminder text when reminder events arrive.

### Health and degraded mode

- The shell displays backend health summary and stage status.
- Text sending, mic usage, quick-add actions, and settings editability are enabled or disabled from health-derived rules.
- Recovery guidance may be injected into the chat transcript by the client.

### Avatar behavior in the current client

- The current client always has placeholder avatar-state handling through `AvatarStateMachine`.
- Audio playback drives subtitle visibility and state changes to talking or idle.
- `LipSyncController` supports amplitude-based motion on a fallback face target.
- If the active Unity scene contains an `AvatarConversationBridge`, the client can also forward listening, speaking, and idle transitions to `Assets/AvatarSystem/`.

## Current Placeholders

These are present in code and should not be documented as finished features:

- schedule center panel calendar grid
- schedule-side assistant recommendation card
- some shell copy such as the `GPT-4.0` chat badge
- production avatar presentation
- mini-assistant window behavior

## Planned Direction

The repo still keeps a design-target document at `Assets/Resources/UI/ui_feature_map.md`.

That file is useful for future polish, but it is not proof that the following exist today:

- a finished premium shell matching the design target
- a compact mini-assistant window
- a fully dynamic right-side schedule insights panel
- a production avatar scene integrated into the shell
- richer click, hover, or floating-avatar behaviors

## Documentation Rule

- Use this document for current UI behavior.
- Use `ui_feature_map.md` only as a design target.
- When the two disagree, code and this document win until implementation catches up.
