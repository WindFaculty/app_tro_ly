# UI - Current Implementation And Planned Direction

Updated: 2026-04-09

This document separates current desktop UI ownership from the retired Unity shell.

## Current Implementation

### Desktop business UI

- Current business UI ownership is the desktop rebuild under `apps/web-ui/` and `apps/desktop-shell/`.
- React owns navigation, business pages, settings, planner, notes, wardrobe management, diagnostics, and chat workspace flows.
- Tauri owns desktop lifecycle, startup gating, runtime status, restore, and packaged-host responsibilities.

### Unity runtime surface

- The live Unity project under `apps/unity-runtime/` no longer ships a business UI Toolkit shell.
- Unity currently owns room composition, avatar presentation, bridge wiring, audio playback, lip-sync support, and runtime-focused tests.
- `AssistantBootstrap`, `StandaloneRoomApp`, and `StandaloneRoomCompositionRoot` now boot the room runtime directly.
- `AvatarStateMachine`, `AvatarRuntime`, `LipSyncController`, `UnityBridgeClient`, and `AvatarConversationBridge` remain part of the live runtime path.

## Current Placeholders

- Production avatar hookup
- Broader room art and prop population
- Richer Unity interaction beyond the current room-focus and runtime bridge surface

## Planned Direction

- React remains the target owner for desktop business UI.
- Unity remains the target owner for room, avatar, camera, and sync-era 3D presentation.
- Older Unity UI docs and UXML references should be treated as historical context, not current implementation.

## Documentation Rule

- Use this document for current ownership and runtime boundaries.
- Do not describe retired Unity shell documents as shipped behavior.
- When docs and code disagree, code wins.
