# UI Spec - Local Desktop Assistant

Updated: 2026-03-13

## Design Direction
- Desktop assistant first, not a generic productivity dashboard.
- Avatar should feel present, but task clarity stays more important than animation detail.
- The app should make `today` and `this week` readable within a few seconds of opening it.
- Runtime problems must surface clearly as `ready`, `partial`, or `error` states.

## Main App Mode
- Layout:
  - avatar area on the left or bottom-left
  - task content in the center
  - chat panel on the right
- Main tabs:
  - `Today`
  - `Week`
  - `Inbox`
  - `Completed`
  - `Settings`

### Core sections
- Avatar panel
  - current state badge
  - subtitle area
  - subtle hover and click reactions
- Task panel
  - grouped task list
  - priority badge
  - due-soon and overdue emphasis
  - quick add entry
- Chat panel
  - message bubbles
  - typing / thinking indicator
  - mic button
  - action feedback after create, complete, or reschedule commands

## Mini Assistant Mode
- Small floating window or compact app mode.
- Primary use cases:
  - open today's agenda quickly
  - ask a short question
  - receive reminder alerts
- Minimum controls:
  - open main app
  - mute speech
  - push-to-talk
  - dismiss reminder

## Voice Mode
- User presses mic.
- UI shows `listening`.
- Transcript preview appears before or during processing.
- UI switches to `thinking`.
- Assistant reply appears as text, then voice playback starts.
- Avatar switches to `talking` and subtitles stay visible during speech.

## Avatar Interaction Rules
- Single click:
  - open quick chat or focus the assistant
- Double click:
  - open the main app window
- Hover:
  - small idle reaction only
- Reminder due:
  - switch to `warning`
  - show popup and optional spoken prompt
- Planner answer:
  - use `serious` or `thinking` emotion with `explain`

## MVP Avatar States
- `idle`
- `listening`
- `thinking`
- `talking`
- `confirming`
- `warning`
- `greeting`

## MVP Lip Sync
- Use audio amplitude from the played output.
- Map amplitude to one mouth-open blendshape.
- Do not block MVP on viseme mapping.

## Settings Surface
- Voice input mode
- TTS voice
- Reply speech on/off
- Mini assistant on/off
- Reminder speech on/off
- Model selection if multiple LLM providers or models are available

## Empty and Error States
- No tasks today:
  - show a calm empty state and suggest quick add
- Backend unavailable:
  - disable task and chat actions and show recovery guidance
- STT unavailable:
  - keep text chat active and disable mic
- TTS unavailable:
  - keep text replies active and suppress playback
